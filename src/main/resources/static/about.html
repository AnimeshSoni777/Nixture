Security Insights

HTTP Security Parameters

WebChecker analyzes server response headers to detect basic, often overlooked layers of defense.

HSTS

Forces browsers to use encrypted HTTPS. Prevents Man-in-the-Middle and Downgrade attacks.

Encryption

CSP

Controls which content sources can be loaded. The main layer of anti-XSS & Injection protection.

Anti-XSS

X-Frame-Options

Prevents websites from being displayed in iframes on other sites. Protects users from clickjacking.

Clickjacking

X-Content-Type

Prevents browsers from MIME-sniffing. Ensures script files are only executed if they are the correct type.

MIME Sniffing

Referrer-Policy

Controls how much referral information is sent when a user clicks an outbound link.

Privacy

Permissions-Policy

Restrict access to advanced browser features to prevent misuse by third-party scripts.

Browser Features

X-XSS-Protection

Legacy header for older browsers. Enables built-in XSS filters to prevent simple scripting attacks.

Legacy

Why Is This Header Important?

Websites without HTTP Security Headers pose a higher security risk. Even if the content is secure, the absence of these headers can be exploited for attacks such as XSS and clickjacking, as browsers lack adequate security restrictions.
